<h1 id="faddom-master-demo-script"># Faddom Master Demo Script</h1>
<h3 id="engineer-to-executive-demo-flow-—-clean-structured-high-value"><em>Engineer-to-Executive Demo Flow — Clean, Structured, High-Value</em></h3>
<hr>
<h1 id="opening">## 0. Opening</h1>
<p>“Thanks for the time. Instead of starting with slides, I’ll show the platform directly.<br>
Everything I present will connect to clear value — visibility, speed, and accuracy.<br>
The platform explains itself and I’ll guide as we go.”</p>
<hr>
<h1 id="dashboard">## 1. Dashboard</h1>
<p><strong>Click:</strong><br>
<code>Dashboard → Toggle Platform Guidance → Show guidance → Hide</code></p>
<p><strong>Say:</strong><br>
“This is the main dashboard. Fully customizable per team — operations, networking, security, migration, asset management.<br>
Platform Guidance explains each screen so the environment is self-teaching.”</p>
<p><strong>Wow effect:</strong><br>
“One unified view of your entire infrastructure without switching tools.”</p>
<p><strong>Avoid:</strong><br>
Long explanations about widgets or settings.</p>
<hr>
<h1 id="search">## 2. Search</h1>
<p><strong>Click:</strong><br>
<code>Search → type a VLAN, subnet, hostname, port, or server</code></p>
<p><strong>Say:</strong><br>
“This is Google for your infrastructure. You can search anything — VLAN, subnet, URL, hostname, server, or port — and jump straight into the live topology.”</p>
<hr>
<h1 id="basic-map-view">## 3. Basic Map View</h1>
<p><strong>Say:</strong><br>
“Each arrow shows real traffic.<br>
The direction shows who initiated the connection — and that immediately reveals the dependency.”</p>
<p><strong>Colors:</strong></p>
<ul>
<li><strong>Green:</strong> Active</li>
<li><strong>Red:</strong> Request sent, no response (firewall or service down)</li>
<li><strong>Gray:</strong> Previously active</li>
</ul>
<p><strong>Wow effect:</strong><br>
“Once we know the direction of a connection, we know which service depends on which.”</p>
<p><strong>Export:</strong><br>
<code>Export → CSV / Visio / Image</code></p>
<hr>
<h1 id="application-map-creation">## 4. Application Map Creation</h1>
<p><strong>Click:</strong><br>
<code>Application Maps → Create &amp; Discover Map → Traffic Flow</code></p>
<p><strong>Steps:</strong></p>
<ul>
<li>Enter entry point (URL, load balancer IP, hostname, etc.)</li>
<li>Select ports</li>
<li>Build layers</li>
<li>Save</li>
</ul>
<p><strong>Say:</strong><br>
“We map the application the same way users access it.<br>
Each purple connection is a newly discovered dependency — real time.”</p>
<p><strong>Avoid:</strong><br>
Over-explaining layers, ports, or settings.</p>
<hr>
<h1 id="change-tracking">## 5. Change Tracking</h1>
<p><strong>Click:</strong><br>
<code>Application Map → Show Change Summary</code></p>
<p><strong>Say:</strong><br>
“This summarizes additions, removals, and new dependencies.<br>
When someone returns from a week away, they immediately see what changed — fully documented.”</p>
<hr>
<h1 id="investigate-anomaly-detection">## 6. Investigate (Anomaly Detection)</h1>
<p><strong>Click:</strong><br>
<code>Pick a server → Charts → Find a peak → Click timestamp</code></p>
<p><strong>Say:</strong><br>
“When traffic behavior changes, we investigate the exact timestamp — who connected, what grew, which new connections appeared.”</p>
<p><strong>Always mention:</strong><br>
“On the right we also see user activity — even zero is meaningful for detecting abnormal access.”</p>
<p><strong>Wow effect:</strong><br>
“Root cause in seconds — no log diving.”</p>
<hr>
<h1 id="impact-set">## 7. Impact Set</h1>
<p><strong>Click:</strong><br>
<code>Server → Add to Impact Set</code></p>
<p><strong>Say:</strong><br>
“If this server goes down, these are the applications that break.<br>
No spreadsheets, no guessing — two clicks.”</p>
<p><strong>Wow effect:</strong><br>
“Eliminates the ‘scream test’.”</p>
<hr>
<h1 id="subnet-dependencies">## 8. Subnet Dependencies</h1>
<p><strong>Click:</strong><br>
<code>Maps → Subnet Dependencies</code></p>
<p><strong>Say:</strong><br>
“This is a bird’s-eye view of all networks — internal east-west and external north-south.”</p>
<p><strong>Examples (required):</strong></p>
<ul>
<li>Group by <strong>production / test / dev</strong></li>
<li>Group by <strong>geography</strong></li>
<li>Group by <strong>data center</strong></li>
<li>Group by <strong>business unit</strong></li>
</ul>
<p><strong>Story example:</strong><br>
“A customer saw immediately that test environment was talking to production — something they never noticed before.”</p>
<p><strong>Wow effect:</strong><br>
“Instant segmentation validation and firewall rule planning.”</p>
<hr>
<h1 id="shadow-it--server-risk">## 9. Shadow IT &amp; Server Risk</h1>
<p><strong>Click:</strong><br>
<code>Security → Shadow IT</code></p>
<p><strong>Say:</strong><br>
“These are servers with no meaningful traffic — leftover projects, test systems, forgotten machines.”</p>
<p><strong>Click:</strong><br>
<code>Security → Servers at Risk</code></p>
<p><strong>Say:</strong><br>
“We combine CVEs, OS end-of-life, software status, exposure, and business impact into a single priority score.”</p>
<p><strong>Wow effect:</strong><br>
“Not just vulnerabilities — actual business impact.”</p>
<hr>
<h1 id="optimization">## 10. Optimization</h1>
<p><strong>Click:</strong><br>
<code>Optimization</code></p>
<p><strong>Say:</strong><br>
“Based on live behavior, we recommend right-sizing both on-prem and cloud resources, so you avoid paying for unused capacity and avoid underprovisioned workloads.”</p>
<hr>
<h1 id="migration-planning">## 11. Migration Planning</h1>
<p><strong>Click:</strong><br>
<code>Migration &amp; DR → Create Wave</code></p>
<p><strong>Say:</strong><br>
“We plan migrations by dependencies.<br>
One server left behind can break an application.<br>
We ensure nothing is missed — cost, network pricing, firewall rules, and dependency validation.”</p>
<p><strong>Wow effect:</strong><br>
“Zero missed servers. Zero broken migrations.”</p>
<hr>
<h1 id="inventory--user-discovery">## 12. Inventory &amp; User Discovery</h1>
<p><strong>Click:</strong><br>
<code>Inventory → Users → Pick user → Open user map</code></p>
<p><strong>Say:</strong><br>
“This shows the user’s real activity — which servers and applications they rely on.<br>
Now you can know who will be affected before taking anything down.”</p>
<hr>
<h1 id="compass-ai">## 13. Compass AI</h1>
<p><strong>Click:</strong><br>
<code>Compass AI</code><br>
<em>(Do not run queries live.)</em></p>
<p><strong>Say:</strong><br>
“If Search is Google for your infrastructure, this is your ChatGPT for your environment.<br>
You can ask:</p>
<ul>
<li>When is the best time to take down a server?</li>
<li>Which users depend on it?</li>
<li>What changed in the last 48 hours?”</li>
</ul>
<p>“All data stays inside your environment.”</p>
<hr>
<h1 id="closing-final-script">## 14. Closing (Final Script)</h1>
<p>“Everything you saw — dependencies, segmentation, drift analysis, optimization, security, users — comes from a single agentless, real-time platform.<br>
It updates continuously, it’s easy to use, and it brings immediate value.<br>
Let me know which part you want to dive deeper into for your environment.”</p>
