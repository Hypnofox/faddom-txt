### 4. USER DISCOVERY / AD INTEGRATION (Section 11)
**Location:** This is its own section - make sure you get here!

**The Action:**
- Show Users in Inventory
- Click on a user
- Open the user map

**What to Emphasize:**
"This is one of the **most underrated features**. Through Active Directory integration, Faddom shows you:

**First:** Which users logged into which servers and when.

**Second:** We map users to the applications they **actually depend on** to do their job.

**Third:** Security anomalies - dormant accounts, unusual access patterns, compromised credentials."

**The Three Value Props:**
1. **Change Management:** "When you take a server down for maintenance, you know exactly which users will be affected. No more surprise calls from the CEO saying 'I can't access my reports.'"

2. **Security:** "See this user who hasn't logged in for 90 days? That's a dormant account that should probably be disabled. See this user logging in from multiple servers at 3 AM? That might be legitimate... or it might be compromised."

3. **Offboarding:** "When someone leaves the company, you see exactly which systems they actually used - not what they were supposed to have access to according to AD groups, but what they actually used."

**Customer Story to Use:**
"A customer discovered a user account that had left the company 6 months ago, still actively logging into servers. Faddom caught the trace. That was a breach waiting to happen."

**Why This Matters:**
- Closes the security visibility gap
- Prevents the "CEO can't access reports" incident
- Compliance and audit gold
- Behavior-based security, not policy-based

**Payoff Line:**
"You can see exactly who will be affected when a server goes offline - bringing it full circle to the 'no more scream test' principle. **Be proactive, not reactive.**"

**CRO Hook:**
Preventing CEO-level incidents = protecting revenue operations

**CTO Hook:**
Security posture and compliance - dormant accounts, unusual access patterns
